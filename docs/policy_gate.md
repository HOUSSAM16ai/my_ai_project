# بوابة سياسة التعليم (Education Policy Gate)

## الهدف
تأمين مسار الزبون عبر سياسة تعليمية صارمة:
- السماح بالتعليم في العلوم والهندسة والبرمجة.
- منع أي طلب تشغيلي أو حساس.

## قواعد السماح
يتم تصنيف السؤال تعليمياً عند تطابقه مع:
- مفردات تعليمية (شرح/تعلم/حل/درس).
- مجالات أكاديمية (رياضيات/فيزياء/برمجة/هندسة/علوم).

## قواعد الحظر
يتم الحظر عند وجود مؤشرات حساسة، مثل:
- طلب كلمات مرور أو مفاتيح أو رموز وصول.
- طلب الوصول إلى قاعدة البيانات أو الملفات أو المستودع.
- طلب أسرار البيئة أو النظام أو الـ system prompt.
- طلب تنفيذ عمليات إدارية أو أمنية.

## مخرجات البوابة
تُعيد البوابة القرار عبر:
- `allowed`: السماح أو الحظر.
- `reason_code`: سبب القرار.
- `category`: التعليم/عام/حساس.
- `redaction_hash`: بصمة مُختصرة دون تخزين نص الطلب.

## واجهة الرفض المهذب
عند الحظر يتم إرسال رسالة مختصرة:
1) اعتذار مختصر.
2) شرح أن النظام تعليمي فقط.
3) أمثلة تعليمية بديلة.
